# 东方财富 AI 分析助手 - 项目章程

## 一、核心目标

打造一个**长期可进化的、个人专用的 A 股多周期 AI 分析工具**。

不是 SaaS,不是个人玩具,是介于两者之间的:
- **工程化**(测试齐全 / 架构清晰)
- **个人化**(只服务我一个人,数据本地化)
- **可进化**(每次使用都让系统变更强,而非越用越烂)

## 二、绝对原则(违反一次终止当前任务)

### 1. "完成"的定义
仅当以下全部满足时,才能声明"任务完成":
- [ ] 代码写完
- [ ] 单元测试覆盖该改动
- [ ] **真实环境跑过该功能**(不是 mock)
- [ ] **完整 console 输出贴在汇报里**
- [ ] 没有 TODO / FIXME / 跳过的测试

仅写完代码 + 测试通过 ≠ 完成。
真实跑过 + 输出验证 = 完成。

### 2. "测试通过"的定义
- 运行 `node --test test/**/*.test.js`(glob 覆盖所有子目录)
- 报告必须包含具体数字:N pass / 0 fail / 0 skip
- 跳过的测试视为失败
- 测试数下降必须有原因说明

### 3. "汇报"的定义
每次任务结束的汇报必须包含:
- 改动文件清单(每个文件做了什么)
- 真实运行截图或 console 输出
- 已知问题清单(哪怕是小问题也写出来)
- 下一步建议

禁止只写"完成"或"测试通过"作为汇报。

## 三、强制工作流(每个任务必须按此顺序)

### 步骤 1:开工前自检
- [ ] 读 .claude-charter.md(本文件)
- [ ] 读 PROGRESS.md 最近 3 个阶段
- [ ] 读 用户给的任务指令
- [ ] 写一段"我理解的任务"反馈给用户确认
- 不写"理解"就开干 = 违反原则

### 步骤 2:计划(plan-before-code)
- 列出要新建/修改的所有文件
- 说明每个改动的理由
- 标出与已有代码的依赖关系
- 标出预期风险

如果计划超过 3 个文件,先给用户看再开干。

### 步骤 3:实施
- 一次只做一件事
- 写完一个文件就跑相关测试
- 不要为了"性能"或"美观"做计划外改动

### 步骤 4:验证(关键!)
真实跑该功能,不是仅跑测试:
- CLI 命令:实际跑 `node cli/index.js <命令>`,贴输出
- 浏览器扩展:在 chrome://extensions 重载,实际打开页面,贴截图
- 数据库:实际 `sqlite3 .eastmoney-ai/db/klines.sqlite` 查数据

### 步骤 5:汇报
按"汇报定义"格式输出。

## 四、特别警告

### 关于浏览器扩展 vs Node CLI
本项目是**双入口**架构:
- 浏览器扩展(Chrome) 用户实时分析
- Node CLI:批量任务 / 自学习 / eval

两个环境**数据通过 Native Messaging 同步**。
任何改动都要考虑两端:
- 改了 lib/ 下任何文件 → 浏览器和 Node 都要跑通
- 不要假设 chrome.* API 在 Node 里能用
- 不要假设 node:fs 在浏览器里能用

### 关于测试 vs 真实
**单元测试通过 ≠ 真实可用**。
反复犯过的错误:
- nightly mock 测试通过,真实 Node 里 chrome.storage 不存在
- scheduled.js 测试报告完成,但文件根本不存在
- financial tool 测试通过,但单位换算错 100 倍

**修复方法**:每个改动写"真实跑通"步骤,而非只写"测试通过"。

### 关于"虚假完成"清单
以下行为视为虚假完成,即使代码看起来在那:
- 报告"完成"但用户提到的 bug 没修
- 报告"通过"但没真实运行
- 报告 N 测试 pass,但 glob 没覆盖所有子目录
- 报告"功能跑通",但 console 输出贴的是几个月前的(从历史复制)
- "我会在下次任务里补全测试" = 当前任务未完成

## 五、用户偏好(覆盖所有默认)

1. **简洁优先**:不要 token 消耗大的回答
2. **结论先行**:理由放后面
3. **自测优先**:Claude Code 自己写测试 + 跑 + 修 bug
4. **直接反对**:不要 sycophantic 开头
5. **预算敏感**:月度 ¥50 限额,DeepSeek 优先批量场景
6. **bug 优于功能**:发现 bug 时立刻停所有新功能,先修 bug
7. **诚实优于完成**:做不完直接说"还有 X% 未完成",不要虚报

## 六、技术栈约定(不允许擅自改)

- LLM:Anthropic Claude(opus-4-7 / sonnet-4-6) + DeepSeek(deepseek-chat)
- Ollama 已废弃,不要重新引入
- 数据源:东财(主) / 新浪 / 腾讯(降级)
- 本地存储:SQLite (better-sqlite3) + JSON 文件
- 测试框架:Node.js native test()
- 不引入新依赖(React/Vue/Express 等)未经用户批准
