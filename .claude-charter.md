# Eastmoney AI Analysis Assistant - Project Charter

## 1. Core Objective

Build a **long-term evolvable, personal A-share multi-period AI analysis tool**.

It is not SaaS, not a toy, but something in between:
- **Engineered** (comprehensive testing / clear architecture)
- **Personalized** (serves only me, data localized)
- **Evolvable** (each use makes the system stronger, not more degraded)

## 2. Absolute Principles (one violation terminates current task)

### 1. Definition of "Done"
Only when ALL of the following are met can a task be declared "done":
- [ ] Code written
- [ ] Unit tests cover the change
- [ ] **Feature tested in real environment** (not mock)
- [ ] **Complete console output included in report**
- [ ] No TODO / FIXME / skipped tests

Code written + tests passing is NOT equal to done.
Real execution + output verification = done.

### 2. Definition of "Tests Passing"
- Run `node --test test/**/*.test.js` (glob covers all subdirectories)
- Report must include specific numbers: N pass / 0 fail / 0 skip
- Skipped tests count as failures
- Test count decrease must have explanation

### 3. Definition of "Report"
Every task completion report must include:
- Change file list (what each file does)
- Real execution screenshot or console output
- Known issues list (even minor ones)
- Next steps suggestion

Prohibited from reporting only "done" or "tests passed".

## 3. Mandatory Workflow (every task must follow this order)

### Step 1: Pre-start Self-check
- [ ] Read .claude-charter.md (this file)
- [ ] Read last 3 phases of PROGRESS.md
- [ ] Read the task instructions from user
- [ ] Write a "My understanding of the task" for user confirmation
- Starting without writing "understanding" = principle violation

### Step 2: Plan (plan-before-code)
- List all files to create/modify
- Explain reason for each change
- Mark dependencies with existing code
- Mark expected risks

If plan exceeds 3 files, show user first before proceeding.

### Step 3: Implementation
- Do one thing at a time
- Run relevant tests after each file
- Do not make unplanned changes for "performance" or "aesthetics"

### Step 4: Verification (Critical!)
Actually run the feature, not just tests:
- CLI: Actually run `node cli/index.js <command>`, paste output
- Extension: Reload in chrome://extensions, open actual page, paste screenshot
- Database: Actually `sqlite3 .eastmoney-ai/db/klines.sqlite` query data

### Step 5: Report
Output in "Report Definition" format.

## 4. Special Warnings

### Browser Extension vs Node CLI
This project has a **dual-entry** architecture:
- Browser Extension (Chrome): User real-time analysis
- Node CLI: Batch tasks / self-learning / eval

Both environments **sync data via Native Messaging**.
Any change must consider both sides:
- Changed any lib/ file -> must work in both browser and Node
- Do not assume chrome.* API works in Node
- Do not assume node:fs works in browser

### Testing vs Reality
**Unit tests passing != actually usable**.
Repeated mistakes:
- Nightly mock test passed, but chrome.storage doesn't exist in real Node
- scheduled.js test reported complete, but the file doesn't exist
- Financial tool test passed, but unit conversion was 100x wrong

**Fix method**: Write "real execution verification" steps for each change, not just "tests passed".

### "False Completion" Checklist
These behaviors are false completion, even if code appears present:
- Reporting "done" but user-mentioned bugs not fixed
- Reporting "passed" but not actually executed
- Reporting N tests pass, but glob doesn't cover all subdirectories
- Reporting "feature works", but console output is months old (copied from history)
- "I'll add tests next task" = current task NOT complete

## 5. User Preferences (overrides all defaults)

1. **Concision first**: No token-heavy responses
2. **Conclusion first**: Reasoning after
3. **Self-test first**: Claude Code writes tests + runs + fixes bugs
4. **Direct objection**: No sycophantic intros
5. **Budget sensitive**: Monthly CNY 50 cap, DeepSeek preferred for batch scenarios
6. **Bugs before features**: Stop all new features on bug discovery, fix bug first
7. **Honesty over completion**: Say "X% unfinished" directly, do not fake

## 6. Tech Stack Conventions (no unauthorized changes)

- LLM:Anthropic Claude(opus-4-7 / sonnet-4-6) + DeepSeek(deepseek-chat)
- Ollama is deprecated, do not reintroduce
- Data sources: Eastmoney (primary) / Sina / Tencent (fallback)
- Local storage: SQLite (better-sqlite3) + JSON files
- Test framework: Node.js native test()
- No new dependencies (React/Vue/Express etc.) without user approval

## 7. Cost of Charter Violations

Each violation will be flagged. 3 consecutive same-type violations = current phase rollback,
redesign.
